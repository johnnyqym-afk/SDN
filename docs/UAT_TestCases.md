# SDN 控制器 - UAT 测试用例框架

## 1. 测试策略与范围

### 1.1 测试层级

| 层级 | 范围 | 覆盖率 | 责任 |
|------|------|--------|------|
| 单元测试 | 单个函数/方法 | >= 60% | 开发人员 |
| 集成测试 | 微服务间交互 | >= 40% | QA工程师 |
| 端到端测试 | 完整业务流程 | 100% | QA + 产品 |
| 性能测试 | 并发/规模/响应时间 | 关键路径 | 性能团队 |
| 安全测试 | 认证/授权/数据安全 | 关键模块 | 安全团队 |

### 1.2 缺陷等级定义

| 等级 | 定义 | 发布阻断 |
|------|------|---------|
| P0/Critical | 系统宕机、数据丢失、无法开通业务 | 是 |
| P1/High | 主要功能不可用、严重影响使用 | 是 |
| P2/Medium | 功能部分受限、影响用户体验 | 否 |
| P3/Low | 界面文本、轻微功能缺陷 | 否 |

## 7. 缺陷管理

### 缺陷报告模板

```
缺陷ID: BUG-001
发现人: 李四
发现时间: 2026-04-07 14:30:00
严重级别: P2
模块: orchestration-service

现象描述:
编排L3VPN时,若VRF名输入超过32字符,系统不拒绝,导致配置下发失败

重现步骤:
1. 进入L3VPN开通向导 (P15)
2. 输入VRF名: "very_very_very_long_vrf_name_exceeds_limit"
3. 点击"确认开通"

期望行为:
系统应在Step 2时提示字符长度限制,不允许超过32字节

实际行为:
系统接受该VRF名,但在发布时预检查报"VRF名格式错误",导致变更失败

影响业务:
业务开通被阻断,需手工修改

建议修复:
前端表单验证补充VRF名长度校验(<=32字符)
```
