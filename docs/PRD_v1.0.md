# SDN 控制器产品设计说明书 v1.0

## 1. 产品目标

1) 将现网的 L3VPN + VPLS + MPLS TE + BFD/NQA 业务开通从"命令行手工"升级为"模板化+流程化+可回滚"。
2) 建立统一资源模型，避免 VLAN/VRF/VSI/隧道策略重复分配与命名漂移。
3) 建立 SLA 保障闭环，实现"预检查-发布-验证-告警-自愈-审计"全流程可追踪。

## 2. 产品范围

1) 纳管 Huawei VRP 设备（优先支持现网同版本设备）。
2) 覆盖业务类型：L3VPN、VPLS/VPWS、TE 隧道策略、QoS 限速策略、BFD/NQA 探测。
3) 覆盖生命周期：新开、扩容、降速、迁移、退网、回滚。
4) 覆盖运营能力：告警、性能、SLA、报表、审计、权限。

## 3. 用户角色

| 角色 | 职责 |

## 6. 模块功能说明

### 6.1 资源中心
- 自动发现设备基础信息、接口、邻居关系、协议状态
- 维护资源池：IP/VLAN/RD/RT/端口库存与占用关系
- 提供冲突检测能力给编排引擎

### 6.2 服务编排中心
- 接收工单参数并绑定标准模板
- 生成业务意图模型，不直接生成命令
- 触发预检查并生成待发布配置候选集

### 6.3 路径与策略中心
- TE 显式路径计算、主备路径约束、重合度控制
- QoS 策略标准化，避免多套"同名不同参"
- SLA 策略标准化，BFD/NQA 参数统一

### 6.4 变更发布中心
- 变更事务化：预检查、分批下发、验收、回滚
- 下发方式优先 NETCONF，兼容 SSH CLI
- 保留发布日志、配置快照、差异对比

### 6.5 运营保障中心
- 汇聚告警、性能指标、SLA结果
- 告警关联业务与路径，支持根因定位
- 触发自动处置策略并闭环回写工单

## 7. 非功能指标（建议作为验收条款）

| 指标 | 目标 |
|------|------|
| 规模 | 500台设备、2万接口、1万业务实例、日均500次变更 |
| 关键列表页响应时间 | P95 < 2秒 |
| 拓扑加载时间 | < 5秒 |
| 控制器集群可用性 | >= 99.95% |
| 自动发布成功率 | >= 98% |
| 单业务回滚时长 | P95 < 5分钟 |
| 安全 | 全链路TLS、审计不可篡改、权限最小化 |
