# SDN 控制器 - 页面级详细设计（32页）

## P01 登录页面

**模块**: 认证  
**功能**: 账号登录、MFA、SSO、租户选择  
**规则**: 连续失败锁定、IP白名单、密码复杂度  
**API**: `POST /auth/login`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | Y | 用户名或邮箱 |
| password | string | Y | 密码（8-20位，需特殊字符） |
| mfa_code | string | N | MFA 6位数字码 |
| tenant_id | string | N | 租户选择（若多租户） |

**规则**:
- 连续5次登录失败，账户锁定30分钟
- IP白名单启用时，非白名单IP拒绝
- 密码复杂度：大小写+数字+特殊字符

## P02 首页工作台

**模块**: 总览  
**功能**: 今日告警、变更进度、SLA红黄绿、资源余量卡片  
**规则**: 数据延迟<60s  
**API**: `GET /dashboard/summary`

### 主要卡片

| 卡片名 | 数据源 | 刷新频率 |
|--------|--------|---------|
| 今日P1告警数 | alarms表 | 实时 |
| 进行中变更数 | change_requests表 | 实时 |
| SLA达标率(%) | sla_results_daily | 每分钟 |
| 设备在线率(%) | devices, heartbeat | 每分钟 |
| 业务实例数 | service_instances | 每小时 |
| 这周变更成功率(%) | deploy_jobs | 每小时 |

## P03 全局搜索页

**模块**: 搜索  
**功能**: 按设备/IP/VRF/VSI/工单号联想搜索  
**规则**: 关键词最少2字符  
**API**: `GET /search?q=<keyword>`

### 支持的搜索类型

- 设备名称 `hostname LIKE %q%`
- 管理IP `mgmt_ip = q`
- VRF名称 `vrf_name LIKE %q%`
- VSI名称 `vsi_name LIKE %q%`
- 工单号 `order_no = q`
- 服务编码 `service_code LIKE %q%`

## P05 告警详情页

**模块**: 告警  
**功能**: 根因链路、关联业务、自动处置建议、操作记录  
**规则**: 仅NOC可关闭P1告警  
**API**: `GET /alarms/{id}`

### 主要内容区

1. **告警基本信息**
   - ID、级别、源类型、首次发生时间、最后发生时间、聚合数

2. **关联业务与路径**
   - 如果是链路上的告警，显示经过此链路的所有服务（拓扑视图）
   - 点击服务可跳转到P18

3. **根因定位**
   - 自动分析相关告警的时间序列，推断根因
   - 例：组合(设备A CPU高+链路B拥塞) → 建议优化路径

4. **自动处置建议**
   - 基于AI模型推荐处置方案
   - 例：P1链路故障 → 自动触发故障路由、拉起工单

5. **操作记录**
   - Who、When、What、Result的完整日志

6. **手工关闭**
   - NOC输入关闭原因和备注
   - 自动生成审计日志

## P07 工单详情页

**模块**: 工单  
**功能**: 需求信息、端点、带宽、SLA、审批流、附件  
**规则**: 必填字段完整后可提交  
**API**: `POST /orders`

### 信息卡片

1. **基本信息**
   - 工单号、申请人、申请时间、客户名称

2. **需求信息**
   - 服务类型、紧急度(1-5)、变更周期(一次性/长期)

3. **端点信息**（A端-Z端模式）
   - A端设备、接口、VLAN、IP地址
   - Z端设备、接口、VLAN、IP地址
   - "点击设备/接口"自动跳转查看拓扑

4. **带宽与SLA**
   - 承诺带宽(Mbps)
   - 可用性目标(%)
   - 延迟目标(ms)
   - 特殊场景(如BGP自带宽、自愈要求)

5. **审批流**

   - 待审批者可点击inline审批/驳回
   - 驳回原因可编辑

6. **附件**
   - 支持上传CAD拓扑、配置参考、需求截图等
   - 文件大小限制100MB

### 提交规则
- A端、Z端、带宽、服务类型必填
- 缺失则按钮置灰并提示缺失项

## P09 设备详情页

**模块**: 资源  
**功能**: 配置快照、接口状态、邻居、BGP/ISIS摘要  
**规则**: 快照版本只读可追溯  
**API**: `GET /inventory/devices/{id}`

### 主要内容区

1. **设备基本信息**
   - hostname、mgmt_ip、vendor、model、version
   - 纳管时间、最后发现时间、最后心跳时间

2. **配置快照**
   - 版本列表（按创建时间倒序）
   - 每个快照显示：时刻、关联变更号、大小
   - 支持对比相邻版本的差异（diff视图）
   - 只读，支持导出

3. **接口列表**
   - if_name、admin_state、oper_state、bandwidth、util%
   - 按接口名排序，可按状态筛选
   - 点击接口名跳转到接口详情/拓扑

4. **邻居关系**
   - LLDP邻居表：local_if -> remote_device/remote_if
   - BGP邻居摘要：neighbor_ip、state、as_num、routes_learned
   - ISIS邻居摘要：neighbor_id、state、metric、level

5. **告警与性能**
   - 最近1h的性能曲线：CPU、内存、磁盘使用率
   - 当前活跃告警列表（link到P04/P05）

**模块**: 资源  
**功能**: L2/L3/MPLS拓扑切换、链路拥塞热力、故障染色  
**规则**: 拓扑刷新周期可配置

### 视图切换

| 视图类型 | 数据源 | 显示内容 |

- 离线设备 → 灰色

## P11 接口资源池页

**模块**: 资源池  
**功能**: 端口库存、占用率、可分配端口推荐  
**规则**: 端口冲突禁止下发  
**API**: `GET /pools/interfaces`

### 资源统计卡

| 卡片 | 公式 | 示例 |
|------|------|------|
| 总端口数 | SUM(物理端口数) | 10,000 |
| 已占用 | SUM(已绑定服务的端口) | 3,500 |
| 占用率 | 已占用/总端口数 | 35% |
| 可用端口 | 总端口数 - 已占用 | 6,500 |

### 端口库存表

| 字段 | 说明 |
|------|------|
| 设备 | hostname |
| 接口 | if_name |
| 类型 | GE/10GE/100GE |
| 速率 | bandwidth(Mbps) |
| 当前状态 | free/allocated/reserved |
| 占用者 | 绑定的service_code |
| 绑定时间 | timestamp |

### 推荐引擎
- 输入："需要在A、B设备间分配一条接口对"
- 输出：推荐可用接口对 + 冲突检查（例：检测是否已被其他工单占用）

## P13 VLAN/标签池页

**模块**: 资源池  
**功能**: VLAN、VNI、RD/RT池配置与冲突检测  
**规则**: 全局唯一约束  
**API**: `GET /pools/labels`

### 标签池类型

| 标签类型 | 范围 | 用途 | 示例 |
|---------|------|------|------|
| VLAN | 1-4094 | 二层隔离 | 1-100(保留), 2000-2999(业务) |
| VNI | 0-16777215 | VXLAN隧道识别 | 5000-5999 |
| RD | 64-bit | VRF路由鉴别器 | AS:value格式 |
| RT | 64-bit | 路由目标 | AS:value格式 |

### 池管理视图

| 列 | 含义 |
|----|------|
| 标签类型 | VLAN/VNI/RD/RT |
| 范围 | 起始-结束 |
| 总数 | 范围内总数 |
| 已分配 | 已使用计数 |
| 占用率 | % |
| 关联业务 | service列表 |

### 冲突检测规则
- **VLAN全局唯一**：同一标签不能分配给两个service
- **RD/RT校验**：import_rt必须在其他VRF的export_rt范围内（确保可递）
- **预告警**：当某类型标签占用率>80%时，在首页告警

## P15 L3VPN开通向导

**模块**: 服务编排  
**功能**: 选站点、选VRF模板、选路由策略、选QoS、选BFD  
**规则**: VRF名唯一、RT冲突校验、默认路由策略校验  
**API**: `POST /provision/l3vpn/plan`

### 向导步骤 (Step 1-6)

#### Step 1: 基本信息
- 服务名 (必填，唯一性校验)
- 客户名 (下拉选择)
- 紧急度 (1-5单选)
- SLA承诺(%) (输入框)

#### Step 2: 站点选择
- 多选站点(设备列表)
- 每个站点需指定：
  - PE设备
  - AC接口
  - AC侧VLAN
  - AC侧IP (可选，自动或手工)
- 支持批量添加后编辑

#### Step 3: VRF模板选择
- 显示已发布的VRF模板列表
- 模板包含：
  - RD (自动或手工input)
  - Import RT列表
  - Export RT列表
- 冲突检查：选定的RT是否与其他VRF冲突 → 警告提示

#### Step 4: 路由策略选择
- "使用默认策略" vs "自定义"
- 默认策略：无过滤，所有PE相互学习
- 自定义：可选route-policy应用于import/export

#### Step 5: QoS配置
- "不启用" vs "启用"
- 启用时选择QoS模板(link到P21)
- 或输入CIR/PIR/CBS/PBS

#### Step 6: BFD探测
- "不启用" vs "启用"
- 启用时选择或输入BFD参数(tx/rx/mult)

### 向导提交

- 点击"确认开通" → 校验所有必填字段
- 后端生成service_order记录、调用编排引擎生成意图模型
- 返回工单号，跳转到P07查看审批状态

---

## P16 VPLS开通向导

**模块**: 服务编排  
**功能**: 选VSI、AC端口、主备隧道、MTU、NQA探测模板  
**规则**: VSI名唯一、端口不能重复绑定  
**API**: `POST /provision/vpls/plan`

### 向导步骤 (Step 1-6)

#### Step 1: 基本信息
- 同L3VPN (service_name, customer, priority, sla%)

#### Step 2: VSI配置
- VSI名称 (唯一性校验)
- VNI (从VNI池推荐)
- RD (从RD池推荐)
- RT (import/export，从RT池推荐)

#### Step 3: AC端口绑定
- 站点1 → AC端口1 + VLAN tag
- 站点2 → AC端口2 + VLAN tag
- ...
- 检查冲突：是否有端口被其他service占用

#### Step 4: 隧道策略选择
- 主隧道：选择TE隧道(link到P19)
  - 或"使用默认MPLS LSP"
- 备隧道：可选
  - 主备模式：1:1保护 / N:1保护
  - WTR时间 (默认300s)

#### Step 5: MTU配置
- 输入L2MTU (默认1500)
- 自动计算MPLS隧道MTU = L2MTU - 18(bytes)

#### Step 6: NQA探测模板
- "不启用" vs "启用"
- 启用时选择NQA模板(link到P22)
- 或手工输入NQA参数(send_freq, timeout, threshold)

### 向导提交
- 同P15流程

## P18 服务实例详情

**模块**: 服务运营  
**功能**: 端到端配置视图、拓扑路径、SLA趋势、关联变更  
**规则**: 变更需创建新版本  
**API**: `GET /services/{id}`

### 主要内容区

#### 1. 服务概览卡片
- service_code、service_name、service_type
- 创建人、创建时间、状态
- 当前版本号、版本历史

#### 2. 端到端配置视图 (按业务类型差异化)

### 模板配置步骤

#### Step 1: SLA指标定义
- 可用性(%), 延迟(ms), 抖动(ms), 丢包率(%)

- Rx间隔(10-1000ms)

- 支持配置多条告警规则，例：
  - 规则1：延迟 > 80ms → P2告警，自动通知NOC
  - 规则2：丢包率 > 1% → P3告警
  - 规则3：连续3个采样点都超过阈值 → 升级到P1

- 审批后发布为生产模板

2. 按批次顺序执行，每batch内设备并发下发
3. Batch完成后等待operator确认或自动进入下一batch

  - 已完成的batch配置自动回滚

- 检查接口、BFD邻接、BGP学习、业务路由是否生效

## P27 回滚中心页

**模块**: 变更  
**功能**: 一键回滚、按任务回滚、手工补偿  
**规则**: 仅发布后24h内自动回滚  
**API**: `POST /rollbacks`

### 回滚策略选择

#### 1. 一键回滚(推荐)
- 列出最近успешные的部署(deploy_job)

- 验证内容(config diff)后点击"确认回滚"
- 系统自动执行逆向配置下发

#### 2. 按任务回滚(精细化)
- 展示该变更单的所有batch与设备
- 用户可选择性地回滚某些batch或某些设备

#### 3. 手工补偿(應急)
- 若自动回滚失败，进入"手工模式"

- 执行完后标记为"MANUAL_INTERVENTION_COMPLETE"

### 回滚执行流程
1. 生成回滚变更单(status=CREATED)
2. 可自动或手工提交审批 (加速路由)
3. 立即进入发布 (不走新变更的冗长审批)

| **接口流量** | in_bps, out_bps, drop_pps, error_pps | 1min | Telemetry/SNMP |
| **隧道时延** | delay_ms, jitter_ms, loss_pct | 60s | NQA或BFD |
| **BFD状态** | 邻接状态(UP/DOWN)、状态变化时间 | 实时 | BFD trap |
| **设备资源** | CPU%, 内存%, 磁盘%, 进程数 | 5min | SNMP |

### 查询与筛选

#### 时间范围选择
- 最近1小时/24小时/7天/30天/自定义日期
- 支持对比：昨天同时段/上周同天/环比增长%

#### 对象筛选
- 选择接口/隧道/设备
- 支持多选
- 支持saved filter(保存常用组合)

### 图表展示

#### 接口流量曲线
- 双Y轴：左轴流量(bps)、右轴丢包率(%)
- 线条颜色区分in/out, 高峰期用不同颜色标记

#### 隧道时延抖动
- 三条线：delay_ms(均值)、95percentile、jitter_ms
- 若有阈值告警，用红线标记告警值

#### 设备资源仪表盘
- 实时CPU%、内存%、磁盘%(圆形进度条)
- 下方列出Top 5进程占用

#### 数据质量指示
- 若某时段缺数据：用灰色区间标注"无数据"
- 原因说明(如采集器DOWN、网络中断等)

### 告警与阈值配置
- 用户可自定义告警阈值(基于SLA模板, P22)
- 超阈值时图表自动高亮并发送通知

## P29 业务SLA看板

**模块**: 运营  
**功能**: 按客户/业务统计SLA达标率、违约明细  
**规则**: 统计口径按自然日/自然月  
**API**: `GET /sla/dashboard`

### 视图模式

#### 1. 客户维度
- 按customer分组统计
- 显示：当月SLA%, 当月违约时长(分钟), 当月违约次数
- 颜色编码：>95%绿, 80-95%黄, <80%红

##### 单客户详情
- 该客户的所有业务实例列表
- 每行：service_name, service_type, current_month_sla%, violations
- 点击service → P18详情

#### 2. 业务维度
- 按service_type分组(L3VPN/VPLS等)
- 显示：平均SLA%, 违约频次, 容量使用率

#### 3. 时间维度
- 日/周/月三种粒度
- 显示历史SLA趋势图表
- 支持对比：本月vs上月/本年vs去年

### 违约明细表

| 列 | 说明 |
|----|------|
| 业务编码 | service_code |
| 违约时段 | 起止时间 |
| 可用性% | 该时段实际可用性 |
| 承诺目标% | 合同SLA目标 |
| 缺陷% | 承诺 - 实际 |

- 违约预警：若本月累计缺陷>某阈值，发红色告警给customer

## P30 报表中心页

**模块**: 运营  
**功能**: 开通时长、故障恢复时长、变更成功率、容量趋势  
**规则**: 报表可追溯到原始数据  
**API**: `GET /reports`

### 报表类型

#### 1. 运营效能报表
- **开通时长分布**
  - X轴：开通时长(hours), Y轴：业务数量
  - 分布：<2h(45%), 2-8h(35%), 8-24h(15%), >24h(5%)
  - 环比数据：本月vs去年同月平均

- **故障恢复时长**
  - P1故障MTTR(Mean Time To Recover)
  - 分业务类型统计
  - 目标：<5min

- **变更成功率**
  - 成功的变更数 / 总变更数 (%)
  - 分变更类型展示(new/expand/migrate)
  - 目标：>=98%

#### 2. 资源容量报表
- **接口利用率趋势**
  - 按PE/CE/RR分组的90day历史趋势
  - 预测：若增长率不变，何时达到85%阈值
  
- **资源池占用率**
  - VLAN池、IP池、RD/RT池的占用率
  - 若某类池>80%，标注预警

- **业务增长趋势**
  - L3VPN实例数、VPLS实例数的月度增长曲线
  - 按customer分组的TOP 10

#### 3. 合规审计报表
- **配置变更审计**
  - 统计周期内的所有变更、变更人、审批人、执行时间
  - 支持按变更类型/风险等级筛选和导出

- **用户操作日志**
  - 登录、配置变更、审批操作
  - 记录actor、操作类型、操作对象、时间戳
  - 支持关键字搜索

### 报表交互

#### 导出
- Excel、PDF多格式
- 支持定时生成与邮件自动化发送(订阅功能)

#### 数据追溯
- 点击图表任意数据点 → 下钻到原始工单/变更/告警记录
- 例：点击"变更成功率95%" → 显示本月所有变更单列表

## P31 审计日志页

**模块**: 治理  
**功能**: 登录、配置、审批、发布、回滚全链路审计  
**规则**: 审计不可篡改、长期归档  
**API**: `GET /audit/logs`

### 日志字段

| 字段 | 示例 | 说明 |
|------|------|------|
| 时间戳 | 2026-04-07 13:45:22 | 精确到秒 |
| 操作人 | alice@example.com | 执行操作的用户 |
| 操作类型 | LOGIN/CREATE/UPDATE/DELETE/APPROVE/DEPLOY | 分类 |
| 对象类型 | user/device/service/change/config | 操作的资源类型 |
| 对象ID | user_5 / service_ABC / change_2026040701 | 资源唯一标识 |
| 操作详情 | 修改服务A的带宽参数(100M→150M) | 具体修改内容 |
| 操作结果 | SUCCESS / FAILURE / PARTIAL | 成功/失败/部分成功 |
| 源IP | 10.1.1.100 | 操作者所在IP |
| 备注 | 紧急变更，通知客户 | 可选说明 |

### 日志列表

#### 筛选条件
- 按操作人
- 按操作类型
- 按对象类型
- 按时间范围
- 按操作结果(成功/失败)

#### 高级搜索
- 支持对象ID模糊搜索
- 支持操作详情全文搜索

### 审计完整性保证

#### 防篡改机制
- 所有日志写入PostgreSQL，主键为(timestamp, actor, object_id)
- 支持：
  - **哈希链**：每条日志含前一条的SHA256哈希，确保不可篡改
  - **长期归档**：日志自动分月份对象存储(MinIO)，只读

#### 导出与报告
- 支持导出为CSV/JSON格式
- 按周/月生成审计报告
- 合规证明：可生成"某时段内此设备的所有配置变更"的完整证明

## P32 系统管理页

**模块**: 系统  
**功能**: 用户角色、南向连接器、发布窗口、参数字典、通知渠道  
**规则**: 改动需二次确认并审计  
**API**: `GET /system/config`

### 子模块设计

#### 1. 用户与权限管理
- **用户列表**
  - 显示：username, email, status(active/locked/disabled), last_login, assigned_roles
  - 操作：新增、编辑、锁定/解锁、重置密码、删除

- **角色管理**
  - 预定义角色：SuperAdmin/NetworkArchitect/ProvisionEngineer/NOCOperator/Approver/Auditor
  - 权限矩阵：行=菜单功能, 列=角色, 交叉点=允许/禁止
  - 支持自定义角色 (仅SuperAdmin)

- **权限分配**
  - 用户 → 角色 → 权限的树状展现
  - 修改后需二次确认(OTP或邮件链接)

#### 2. 南向连接器配置
- **连接器清单**
  - 列表：connector_type(NETCONF/SSH_CLI/SNMP), status(active/inactive), last_sync_time
  - 操作：新增、编辑、测试连接、启用/禁用、删除

- **单个连接器详情**
  - 连接方式：协议(SSH/Telnet)、对应端口、超时设置
  - 认证方式：用户/密码、密钥对(支持密钥存储于Vault)
  - 关联设备范围：支持按vendor/model/version过滤应用此连接器的设备
  - 健康检查：定期测试连接，失败则告警

#### 3. 发布窗口管理
- **窗口列表**
  - 显示：window_name, schedule(cron格式), start_time, end_time, capacity(最多并发变更数)
  - 例：周五22:00-周六06:00, 支持10个并发变更

- **窗口编辑**
  - 名称、时间表(单次/重复)、容量
  - "应用自动发布规则"：选中窗口内的变更自动发布(无需手工确认)
  - 修改后需二次确认

#### 4. 参数字典
- **K-V列表**
  - 常用参数：max_batch_size(默认5), precheck_timeout(默认300s), bfd_default_tx(默认100ms), nqa_default_freq(默认60s) 等
  - 支持按分类(网络/系统/告警)筛选

- **参数编辑**
  - 输入参数值，系统校验合法性(数值范围、格式等)
  - 修改后自动刷新所有关联的业务逻辑(无需重启)

#### 5. 通知与告警渠道
- **渠道配置**
  - 支持类型：邮件、短信、钉钉、企业微信、Webhook
  - 每个渠道配置：接入点、账号、频率限制

- **规则配置**
  - 告警级别 → 对应发送渠道
  - 例：P1告警一律发短信+钉钉; P3以下仅邮件
  - 接收人分组：可按team/role自动路由

#### 6. 日志与诊断
- **系统日志**
  - 访问日志、错误日志、慢查询日志(列表与搜索)
  - 支持导出与分析

- **性能诊断**
  - DB连接池状态、缓存命中率、消息队列深度(仪表板)
  - 支持手工触发诊断报告生成
